# 个人影音库

> 以「编号 + NFO」为核心的本地个人影音库。前端 React + TypeScript + Chakra UI，后端 Flask + SQLite，通过浏览器在局域网中浏览、筛选和播放本地视频。
>
> 核心设计是：**你已经有一堆按编号+NFO 管理好的本地视频，希望有一个更顺手、更“懂你”的浏览与检索工具，而不是一套庞大复杂的家庭媒体中心。**

---
## 与主流媒体库的差异

- **更“轻”的目标**
    - 不追求「全家人共用的大型媒体中心」，所有的视频都是面向 **一个人看片需要的操作习惯，所有功能都是我自己用得上的**。
    - 没有复杂的多用户、权限、远程访问和转码集群，部署和使用都尽量保持简单。
- **更尊重实际体验的视频管理与播放**
    - 支持为视频自动从网络匹配字幕！网络资源经常因为广告字幕有时间差，所以支持方便地基于台词调整字幕时间偏移！
    - 支持为视频添加书签，不会放过神片的每一个顶级画面！
    - 海报墙支持更灵活的检索，包括seed随机/复合Tag。Tag就是片单。不再像Jellyfin一样tag检索不方便，且和片单是完全两套管理逻辑！
    - 更棒的视频预览，我愿称之为真正适合海报墙的卡片式视频预览（用了就知道了）
- **尊重你现有的整理习惯**
    - 不强行改文件命名规则，也不强行在线刮削，只要你已经有 NFO，就按你的 NFO 来展示。
    - 修改类型/标签时可以写回 NFO，让你的「文件即真相」模式继续成立。

---
## 你能用它做什么？

- **与主流媒体库兼容，把现有的编号+NFO 视频库“一键变成”可视化影音库**
  - 识别形如 `CODE.nfo` + `CODE.mp4/ts` 的文件对，自动导入为条目。
  - 按列表、封面墙方式浏览，点击即可查看详情与播放。
  - 资源库的管理更简单，没有多库，可以配置多个资源路径，最终会统一记录在一个库里

- **更符合实际需要的检索方式**
  - 搜索框支持按 **编号 / 标题** 直接搜。
  - 类型（Genre）+ 标签（Tag）可以任意组合过滤；每个选项会显示“目前库里有多少条用了这个标签”。
  - 筛选模式支持 **交集（AND）/ 并集（OR）** 一键切换，轻松表达「又是 A 又是 B」或「A 或 B 都行」的需求。

- **获得更好的播放与预览体验**
  - 内置 Video.js 播放器，支持本地文件和 HLS（m3u8）。
  - 进度条上有 **雪碧图缩略图预览**（基于自研 WebVTT 插件），拖动时可以更直观地“找位置”。

- **按“演员视角”重新认识自己的库**
  - 条目详情中会列出演员，后端维护独立的演员索引库。
  - 可选连接一个外部头像仓库（提供 Filetree 索引），自动为库内演员拉取头像并缓存在本地，在侧栏/详情页展示更友好的演员卡片。

## 核心功能一览

- **NFO 驱动的媒体库**
  - 「编号 + `.nfo`」是唯一元数据来源，同目录自动匹配 `code.mp4` / `code.ts`。
  - 扫描后将元数据与文件信息缓存到 SQLite，用于快速列表、筛选与统计。

- **多路径媒体库与扫描控制**
  - 在前端「数据源配置」页面管理多个媒体根路径。
  - 支持「启动时自动扫描」与「手动一键扫描」两种模式。
  - 扫描时跳过模板 NFO（如 `movie.nfo` / `template.nfo` 等），并对重复编号去重。

- **强大的筛选与排序**
  - 编号 / 标题搜索。
  - 类型 + 标签组合筛选，带使用计数。
  - AND / OR 筛选模式切换，一键清空过滤器。
  - 排序支持按编号、按时间与随机；随机模式带种子写入 URL，同一个链接始终得到相同的随机顺序。

- **播放器与预览体验**
  - 统一使用 Video.js 播放器，内部用 hls.js 处理 HLS。
  - 自研 WebVTT + 雪碧图进度条缩略图，拖动进度条即可预览画面。

- **演员与图片系统**
  - 维护「编号 - 演员」关系，可按演员查看相关作品。
  - 支持配置外部演员头像仓库，自动为库内演员挑选和缓存头像。
  - 条目详情优先使用本地缓存头像，无则回退到 NFO 中配置的缩略图。

- **NFO 为中心的数据模型**
  - 所有媒体数据都以 NFO 为准，数据库只是缓存与索引。
  - 提供写回 NFO 的类型/标签更新能力，方便以后做前端元数据编辑，同时保证 NFO 和库中的显示保持一致。

## 快速开始

### 运行要求

- Python 3.10+
- Node.js 18+ / npm
- 支持访问媒体根目录和数据库文件的本地磁盘或网络路径

### 启动后端

```bash
python -m backend.app
```

- 默认监听：`http://0.0.0.0:5000`
- 首次启动会根据配置自动建库并可选执行一次全库扫描。

### 启动前端

```bash
cd frontend
npm install
npm run dev
```

- 默认通过 Vite 开发服务器运行，`/api` 请求通过代理转发到后端。

### 默认账号

- 用户名：`admin`
- 密码：`admin`

可通过环境变量覆盖（见下方「配置」）。

## 基本使用流程

1. **配置媒体库路径**
   - 登录后进入「数据源配置」页面，添加一个或多个媒体库路径（如 `D:\Media`、`/mnt/media`）。
   - 选择是否在后端启动时自动执行扫描（推荐开启）。

2. **准备 NFO 与视频文件**
   - 在媒体库路径下按「编号」组织资源，例如：
     - `SENPAI-114514.nfo`
     - `SENPAI-114514.mp4`（或 `SENPAI-114514.ts`）
   - NFO 中包含标题、简介、类型、标签、演员等元数据。

3. **扫描媒体库**
   - 若开启自动扫描，后端启动时会遍历所有媒体库路径并导入 NFO。
   - 也可以在前端点击「刷新库」按钮，调用 `POST /api/scan` 手动触发扫描。

4. **浏览与筛选**
   - 在媒体库列表页按编号/标题搜索条目。
   - 使用类型/标签过滤器组合筛选，并根据需要切换交集/并集模式。
   - 切换排序方式（按编号 / 按时间 / 随机），URL 会随之更新，方便刷新或分享。

5. **查看详情与播放**
   - 点击条目进入详情页，查看 NFO 提供的完整元数据、封面、演员列表等。
   - 点击「播放」进入播放页，使用 Video.js 播放器播放本地视频，进度条支持缩略图预览。

## 配置一览

- **环境变量（后端，常见项）**
  - `MEDIA_ROOT` / `DB_PATH` / `SECRET_KEY` 等基础项。
  - 详情与默认值见 `docs/PLAN_AND_ROADMAP.md` 中「配置与环境变量」。  

- **前端「数据源配置」页面可视化项**
  - 媒体库路径列表（`media_roots`）。
  - 启动时自动扫描开关（`scan_on_startup`）。
  - 演员头像仓库 URL（`avatar_source_url`，留空即关闭头像同步）。
